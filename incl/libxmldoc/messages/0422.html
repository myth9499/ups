  Ok this is the code I have been working on over the last 2 months.
It breaks binary compatibility, and currently it's a beta release.
 It is available from:

   ftp://rpmfind.net/pub/libxml/

as tar.gz and RPMs. You may have to force it a bit to coexist with
libxml-1.x but it's doable :-) . The old devel should be removed and
replaced.

This includes a very large set of changes. From a programming point
of view applications should not have to be modified too much, check
the upgrade page
   http://xmlsoft.org/upgrade.html

Some interfaces may still changes (especially a bit about encoding).
the updates includes:
 - fix I18N support. ISO-Latin-x/UTF-8/UTF-16 (nearly) seems
    correctly handled now
 - Better handling of entities, especially well formedness checking
    and proper PEref extensions in external subsets
 - DTD conditional sections
 - Validation now correcly handle entities content
 - change structures to accomodate DOM
 - Serious progress were made toward compliance, here are the result
   of the test against the OASIS testsuite (except the japanese tests
   since I don't support that encoding yet). This URL
   is rebuilt every couple of hours using the CVS head version.

   http://xmlsoft.org/conf/result.html

 One of the main things still generating problem is bad/incorrect
standalone flag checking and handling. There is still an extension needed
to handle complicated UTF-16 inputs.

  Please try it, and report problems,

Daniel
