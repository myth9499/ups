> I found that xmlKeepBlanksDefault(0) doesn't work in 2.3.3 now. Any one has

   Hum, no.

 xmllint option '--noblanks' exercise this, and as far as I can tell
it still seems to work both for the CVS version and for xmllint as released
in 2.3.3 .

A simple way to test is to run
    xmllint --debug --noblanks testfile
and
    xmllint --debug testfile

and verify that the 'superfluous' white space nodes have been stripped
