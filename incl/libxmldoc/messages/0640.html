The --without-zlib option for configure doesn't do anything. libxml
gets zlib support if configure can find the header regardless of the
setting of this option. I'm not using it and it causes build problems
for my users (I have LDFLAGS=-lz in my makefile, this fails on boxes
without the zlib static library). I suppose one answer is to fix my
Makefile.am and configure.in. Another is for libxml's configure to
not accept the --without-zlib option, it's misleading.
