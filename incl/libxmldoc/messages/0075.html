On Tue, Jun 22, 1999 at 08:07:43AM -0500, Miguel de Icaza wrote:
> Yes.
>
> Now, the big problem we are facing here is that package managers like
> RPM are not designed to have you keep multiple versions of the same
> library around. You can not have libxml-0.1 and libxml-0.2 installed
> on the system unless you resort to some spec hacking and magic (like
> gtk10, gtk rpms).

Actually, if you use the 'install' feature instead of the 'upgrade'
feature in RPM you can have two version of the same package installed
at the same time.

[root@erwin i386]# rpm -q kernel
kernel-2.2.5-23
kernel-2.2.5-15

Not saying that this is a good thing to do in this case, mind you.
Binary backward compatibility should be maintained.
