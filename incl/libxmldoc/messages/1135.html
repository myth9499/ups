The attached patch adds a new item to the end of the xmlNanoHTTPCtxt
structure that holds the WWW-Authorization or Proxy-Authorization
header contents. I also added the function xmlNanoHTTPAuthHeader() to
fetch this value from the void ctxt pointer.

With these changes it is now possible to find out what "realm" the 401 or
407 error is in, making it possible to leverage password information for
common realms (on the same site, of course).
